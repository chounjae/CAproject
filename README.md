#git rule

<hr>

github에 있는 변경사항 가져오기

git pull origin main

<hr>

변경 사항 커밋 준비하기(커밋 전 필수)

git add . (. 은 변경한 파일 전부를 의미 [파일명]으로 하나만 커밋 가능)

<hr>

커밋 메시지

git commit -m

<hr>
  
변경 사항 푸시하기

git push origin main

<hr>

새로운 브랜치 만들기

git checkout -b 새로운브랜치이름

<hr>

현재 브랜치 확인

git branch

<hr>

기존 브랜치로 이동

git checkout 브랜치이름

<hr>

브랜치 삭제 

git branch -d 브랜치이름
